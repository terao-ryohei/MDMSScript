# 統合テストシナリオ設計書

## 1. ゲームライフサイクルテスト

### 目的
ゲームの開始から終了までの一連の流れが正常に動作することを確認する。

### 前提条件
- サーバーが起動している
- 必要な最小人数のプレイヤーが参加している
- 全てのマネージャーが初期化されている

### テストシナリオ

#### 1.1 ゲーム開始フロー
1. ゲームを開始する
2. 各プレイヤーに役職が割り当てられる
3. 初期フェーズに移行する

**検証ポイント**
- GameManagerが正しく初期化される
- 全プレイヤーに役職が割り当てられる
- PhaseManagerが正しく初期フェーズを設定する

#### 1.2 フェーズ遷移
1. 各ゲームフェーズを順番に実行
2. フェーズごとの制限時間を確認
3. フェーズ終了条件の検証

**検証ポイント**
- フェーズが正しい順序で遷移する
- 各フェーズで適切なアクションのみが許可される
- フェーズ終了時に正しく次のフェーズに移行する

#### 1.3 役職機能
1. 各役職固有のアクション実行
2. 役職間の相互作用確認
3. 役職情報の秘匿性確認

**検証ポイント**
- 各役職が正しい権限とアクションを持つ
- 役職情報が適切に管理される
- 役職間の相互作用が想定通りに機能する

## 2. 証拠システムテスト

### 目的
証拠の収集、分析、共有機能が正しく動作することを確認する。

### テストシナリオ

#### 2.1 証拠収集
1. プレイヤーが証拠を発見
2. 証拠の種類に応じた収集アクション
3. インベントリへの追加

**検証ポイント**
- 証拠が正しく収集できる
- 証拠の種類が正しく識別される
- インベントリシステムが正常に動作する

#### 2.2 証拠分析
1. 収集した証拠の分析実行
2. 分析結果の表示
3. 証拠の関連付け

**検証ポイント**
- EvidenceAnalyzerが正しく機能する
- 分析結果が適切に表示される
- 証拠間の関連性が正しく検出される

#### 2.3 証拠共有
1. プレイヤー間での証拠共有
2. 共有された証拠の検証
3. 証拠チェーンの構築

**検証ポイント**
- 証拠が正しく共有される
- 共有された証拠の整合性が保たれる
- 証拠チェーンが正しく構築される

## 3. コミュニケーションテスト

### 目的
プレイヤー間のコミュニケーション機能が正しく動作することを確認する。

### テストシナリオ

#### 3.1 メッセージング
1. プレイヤー間でのメッセージ送信
2. グループチャットの機能
3. フェーズごとの通信制限

**検証ポイント**
- メッセージが正しく送受信される
- チャットの可視性が正しく制御される
- フェーズに応じた通信制限が機能する

#### 3.2 投票システム
1. 投票の開始と終了
2. 投票結果の集計
3. 投票結果に基づくアクション実行

**検証ポイント**
- 投票プロセスが正しく実行される
- 投票結果が正確に集計される
- 投票結果に基づくアクションが正しく実行される

#### 3.3 情報共有
1. 役職に応じた情報共有
2. チーム内での情報共有
3. 共有情報の制限確認

**検証ポイント**
- 役職に応じた情報が適切に共有される
- チーム内での情報共有が正しく機能する
- 情報の可視性が適切に制御される

## 4. ActionLogger統合テスト

### 目的
アクションのログ記録と分析機能が正しく動作することを確認する。

### テストシナリオ

#### 4.1 アクションログ記録
1. 各種ゲームアクションの実行
2. ログの記録確認
3. ログフォーマットの検証

**検証ポイント**
- 全てのアクションが正しくログに記録される
- ログのフォーマットが仕様通りである
- タイムスタンプが正確に記録される

#### 4.2 ログフィルタリング
1. 種類別のログフィルタリング
2. 時間範囲でのフィルタリング
3. プレイヤー別のフィルタリング

**検証ポイント**
- フィルタリング条件が正しく機能する
- フィルタリング結果が正確である
- パフォーマンスが要件を満たす

#### 4.3 ログ分析
1. アクションパターンの分析
2. 時系列分析
3. プレイヤー行動分析

**検証ポイント**
- 分析機能が正しく動作する
- 分析結果が有用な情報を提供する
- 分析パフォーマンスが要件を満たす

## 統合テスト実行環境

### 必要な環境設定
- Minecraft サーバー
- テストプレイヤーアカウント（最低4アカウント）
- ActionLogger設定
- テストデータセット

### テスト実行の注意点
1. 各テストケース実行前に環境をリセット
2. テストデータの整合性を確認
3. エラーケースの検証も含める
4. パフォーマンス指標の測定
