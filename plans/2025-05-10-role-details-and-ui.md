# 役職詳細設計とUI表示仕様

## 1. 役職の詳細定義

### 探偵 (Detective)
- **目的**: 殺人者と共犯を見つけ出し、他のプレイヤーと協力して事件を解決する
- **特殊能力**:
  - 証拠の信頼性スコアを確認できる
  - 1日に1回、特定のプレイヤーの行動ログを詳細に分析できる
- **勝利条件**: 
  - 殺人者が投票で追放される
  - または、殺人者が証拠によって有罪となる
- **その他**:
  - 常に証拠の信頼性スコアが表示される
  - 分析結果は他のプレイヤーと共有可能

### 殺人者 (Killer)
- **目的**: 探偵と市民を欺き、罪を逃れる
- **特殊能力**:
  - 1日に1回、プレイヤーを殺害可能
  - 偽の証拠を作成可能（ただし信頼性は低い）
- **勝利条件**:
  - 探偵が投票で追放される
  - または、無実のプレイヤーが有罪となる
- **その他**:
  - 共犯者とのプライベートチャットが可能
  - 殺害は夜間フェーズのみ可能

### 共犯 (Accomplice)
- **目的**: 殺人者を支援し、探偵と市民を欺く
- **特殊能力**:
  - 殺人者とのプライベートチャットが可能
  - 偽のアリバイを作成可能
- **勝利条件**:
  - 殺人者と同じ
- **その他**:
  - 殺人者の正体を知っている
  - 殺害能力は持たない

### 市民 (Citizen)
- **目的**: 探偵と協力して殺人者を見つけ出す
- **特殊能力**:
  - 基本的な証拠収集能力
  - 他のプレイヤーとの会話機能
- **勝利条件**:
  - 探偵と同じ
- **その他**:
  - 最も数が多い役職
  - チームプレイが重要

## 2. UI表示設計

### 2.1 役職情報表示のタイミング

1. **ゲーム開始時**:
   - 全画面オーバーレイで役職を表示
   - 役職の基本情報（目的、勝利条件）を表示
   - 確認ボタンをクリックで次の画面へ

2. **ゲーム中**:
   - 画面右上に常時表示される役職アイコン
   - アイコンホバーで簡易情報を表示
   - アイコンクリックで詳細情報を表示

3. **フェーズ切り替え時**:
   - 役職特有のアクションが可能な場合、通知を表示
   - 例：殺人者の夜間フェーズ開始時など

### 2.2 レイアウト構成

#### メイン画面レイアウト
```
+------------------+
|  役職アイコン    |  <- 常時表示
+------------------+
|                  |
|    ゲーム画面    |
|                  |
+------------------+
```

#### 役職詳細画面レイアウト
```
+----------------------------------+
|  役職名 + アイコン               |
+----------------------------------+
|  目的                            |
+----------------------------------+
|  特殊能力                        |
|  - 能力1                         |
|  - 能力2                         |
+----------------------------------+
|  勝利条件                        |
+----------------------------------+
|  現在の特殊能力の状態            |
|  [ クールダウンタイマー等 ]      |
+----------------------------------+
```

### 2.3 インタラクション設計

1. **役職アイコン**:
   - ホバー: 簡易情報をツールチップで表示
   - クリック: 詳細情報画面を表示
   - 右クリック: コンテキストメニュー表示（可能な場合）

2. **特殊能力UI**:
   - 使用可能時にハイライト表示
   - クールダウン時はタイマー表示
   - 使用不可時は半透明表示

3. **通知システム**:
   - 重要な役職関連イベントはポップアップで通知
   - 優先度に応じて表示位置とスタイルを変更
   - 一時的な通知は自動で消滅

### 2.4 アクセシビリティ考慮事項

1. **カラーコーディング**:
   - 各役職に固有の色を割り当て
   - 色覚異常者に配慮した配色
   - アイコンと組み合わせて識別性を確保

2. **テキストとコントラスト**:
   - 重要情報は十分な文字サイズで表示
   - 背景とのコントラスト比を確保
   - フォントは可読性の高いものを使用

3. **キーボード操作**:
   - すべての機能をキーボードで操作可能
   - ショートカットキーの設定
   - フォーカス表示の視認性確保

## 4. 技術的考慮事項

1. **状態管理**:
   - 役職情報をGameStateで管理
   - UIの状態をローカルで管理
   - 特殊能力の状態を同期

2. **パフォーマンス**:
   - 不要な再レンダリングを防止
   - アニメーションの最適化
   - メモリ使用量の監視

3. **セキュリティ**:
   - 役職情報の暗号化
   - チート防止措置
   - 通信の安全性確保
